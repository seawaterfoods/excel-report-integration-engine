需求規格書：Excel 數據自動化整合工具
一、 系統輸入參數 (Configuration)
程式需提供設定介面（或設定檔），供使用者定義以下參數：

讀取模式設定：需支援以下幾種選取方式：

1.區域讀取：指定起始單元格（如：B4），自動抓取該點之後所有具資料之橫向與縱向欄位。

2.橫向讀取：指定起始單元格至結束單元格（如：B4 至 Z4）。
 .縱向讀取：指定起始單元格至結束單元格（如：B4 至 B24）。
 
3.缺值填補：可以讀取各匯入檔中有參數部分，自動填補於匯出檔中，該單元格中有空值就填入

注意:若有多匯入檔有重複填值，需要紀錄該問題並停止匯出檔案

預期匯入總數：設定本次作業應匯入的檔案總量（例如：19 份）。

二、 處理邏輯 (Processing Logic)
範例檔[樣板]繼承：產出之結果檔需完全沿用「範例檔」的格式與內容，且必須保留原有的 Excel 公式（不可僅貼上值），不可變更範例檔中原有格式及內容，僅能填入空值，理論上各匯入檔會包含樣板中參數，但匯入檔會有各公司各自獨有資料。

資料寫入規則：
程式讀取來源檔參數範圍內的數值後，直接寫入產出檔對應位置，不進行任何格式轉換或運算。

自動命名規則：
產出檔名邏輯：[執行當日西元年月日]_[原檔名底線後的字串]。
範例：匯入檔名為 01_經驗收集表，產出檔名則為 20260427_經驗收集表.xlsx。

指定讀取頁籤：
程式僅讀取匯入檔中的第一個工作表（First Sheet / Index 0）。
不論該頁籤名稱為何，皆以索引順序為準。若匯入檔包含多個頁籤，其餘頁籤內容一律忽略不計。

三、 檢核機制
程式執行時需進行以下自動化檢查，若不符則跳出錯誤提示：

檔案數量檢核：
檢查實際匯入的檔案數量是否等於「預期匯入總數」參數。若數量不符（多於或少於），則中止程序。

檔案名稱格式：
匯入檔案名稱的前兩碼必須為數字（00-99），如：01_XXX統計表。

資料讀取不重複檢核：
程式需記錄已讀取的單元格座標。
跨檔案檢核：若第一份檔案已讀取了 B4、B5 範圍，後續檔案的讀取範圍設定不得包含已讀取過的座標，避免資料重疊或覆蓋。

## 3. 目錄結構

```
project-root/
├── import/                    # 放入來源 xlsx 檔案
│   └── 115/                   # 民國年
│       ├── 01/
│       │   ├── templet/XXX.xlsx
│       │   ├── 01_XXX.xlsx
│       │   └── 05_XXX.xlsx
│       └── 02/
│           ├── templet/XXX.xlsx
│           └── 29_XXX.xlsx
├── output/                    # 系統自動產出報表
│   ├── 115/
│   │   └── 01/
│   │       └── XXX.xlsx
│   └── 114/                   
│       └── 02/
│           └── XXX.xlsx
├── config/                    # 外部設定檔
│   └── application.yml
└── (程式碼目錄)
```


外部設定檔位於 `config/application.yml`：

```yaml
app:
  import-dir: ./import                         # 來源檔案目錄
  output-dir: ./output                         # 輸出目錄
  process-year: 115                            # 處理年度（民國年，Integer）
  process-month: 01                            # 處理月（月，Integer）
```
僅處理該年月報表
若沒設定月但有設定年，就處理設定該年的所有報表
如果沒設定年，就處理所有報表
如果沒設定年但有設定月，就處理每年該月的報表
