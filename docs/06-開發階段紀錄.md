## Phase 4：FileService ✅

**Commit**: `feat: implement core services and orchestrator`

### 核心功能

- `resolveReportTasks()` — 根據 year/month/report 設定掃描目錄，產出 ReportTask 清單
- 支援新目錄結構 (`year/month/reportName/templet/`) 與舊結構 (`year/month/templet/`) 向後相容
- `loadScopeConfig()` — 讀取 `templet/scope.yaml` 或 `scope.yml`，預設 AUTO_DETECT
- `findTemplateFile()` — 找第一個 .xlsx (排除 ~ 暫存檔)
- `findImportFiles()` — 列出報表目錄中所有 .xlsx

---

## Phase 5：ExcelService ✅

**Commit**: (同 Phase 4)

### 核心功能

- `openWorkbook()` — XSSFWorkbook 載入 (保留公式)
- `copyTemplate()` — Files.copy 完整複製 (保留所有工作表/格式/公式)
- `readAllCellData()` — 遍歷 Sheet[0] 非空 cell，轉為 CellData
- `writeCellData()` — 將 CellData 寫入指定工作簿 (僅寫值不改格式)
- `parseCoordinate()` — 座標字串轉 [row, col] (0-based)

**設計決策**: 公式 cell 不作為「資料」讀取，保留原始公式不受影響。

## Phase 6：ValidationService ✅

**Commit**: (同 Phase 4)

### 檢核規則

1. `validateFileCount()` — 比對 expectedFileCount (null = 跳過)
2. `validateFileNames()` — 正規表達式 `^\d{2}_.+\.xlsx$`
3. `validateNoDuplicates()` — Map<座標, 來源檔> 追蹤，偵測跨檔重複

---

## Phase 7：MergeService ✅

**Commit**: (同 Phase 4)

### 萃取邏輯

- 建立樣板 Map<座標, 值>
- 遍歷匯入檔 cell，與樣板比對
- 座標不存在或值不同 → 獨有資料 → 加入待寫入清單

---

## Phase 8：ReportOrchestrator ✅

**Commit**: (同 Phase 4)

### 執行流程

1. 載入設定 → 解析報表任務
2. 逐一處理 (各報表獨立)
3. 前置檢核 → 複製樣板 → 逐檔讀取+比對+檢核重複 → 寫入 → 儲存
4. 產出 JSON 執行報告 (`logs/report.json`)
5. 重複座標時中止該報表並刪除不完整輸出
