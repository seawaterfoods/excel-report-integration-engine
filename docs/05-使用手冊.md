# 使用手冊

> 專案：Excel Report Integration Engine v1.0  

---

## 一、系統概述

本工具自動讀取多份 Excel 匯入檔，將各公司獨有資料整合至樣板檔，產出合併後的報表。

### 功能特色

- 自動比對樣板與匯入檔，萃取獨有資料
- 支援多年/月/報表批次處理
- 完整檢核機制 (數量、檔名格式、重複座標)
- 保留樣板所有格式與公式
- JSON 執行報告
- 支援本地與 Docker 兩種執行方式

## 三、目錄結構

```
project-root/
├── import/                          # 來源檔案
│   └── {年}/                        # 民國年 (如 115)
│       └── {月}/                    # 月份 (如 01)
│           └── {報表名}/            # 報表名稱
│               ├── templet/         # 樣板目錄
│               │   ├── 報表.xlsx    # 樣板檔
│               │   └── scope.yaml   # 讀取設定 (選用)
│               ├── 01_報表.xlsx     # 匯入檔
│               ├── 02_報表.xlsx
│               └── ...
├── output/                          # 自動產出
│   └── {年}/{月}/
│       └── 20260427_報表.xlsx
├── config/
│   └── application.yml              # 主設定檔
└── logs/
    ├── engine.log                   # 執行日誌
    └── report.json                  # 執行報告
```

## 四、設定說明

### 4.1 主設定檔 (`config/application.yml`)

```yaml
app:
  import-dir: ./import               # 來源檔案根目錄
  output-dir: ./output               # 輸出根目錄
  process-year:                      # 民國年 (留空=全部)
  process-month:                     # 月份 (留空=全部)
  process-reports:                   # 報表名，逗號分隔 (留空=全部)

logging:
  level:
    com.excel.engine: INFO           # 日誌等級: DEBUG / INFO / WARN / ERROR
  file:
    name: ./logs/engine.log
```

### 篩選邏輯

| process-year | process-month | process-reports | 行為 |
|:---:|:---:|:---:|------|
| 有 | 有 | 有 | 僅處理指定年/月/報表 |
| 有 | 有 | 空 | 處理指定年月的所有報表 |
| 有 | 空 | 空 | 處理指定年的所有月份 |
| 空 | 有 | 空 | 處理所有年份的指定月份 |
| 空 | 空 | 空 | 處理全部 |

### 4.2 報表讀取設定 (`templet/scope.yaml`，選用)

```yaml
read-mode: AUTO_DETECT               # AUTO_DETECT | AREA | HORIZONTAL | VERTICAL
start-cell: B3                       # 起始儲存格 (AREA/HORIZONTAL/VERTICAL)
end-cell: H22                        # 結束儲存格 (HORIZONTAL/VERTICAL)
expected-file-count: 20              # 預期匯入數 (留空=不檢核)
```

- 若不存在此檔案，預設使用 `AUTO_DETECT` 模式 (自動比對樣板差異)
- `expected-file-count` 為各報表獨立設定

## 五、執行方式

### 5.1 本地執行

```bash
# 編譯
mvn clean package -DskipTests

# 執行
java -jar target/excel-report-integration-engine-1.0.0-SNAPSHOT.jar \
  --spring.config.location=file:./config/application.yml
```

### 5.2 Docker 執行

```bash
# 建構映像
docker-compose build

# 執行
docker-compose up

# 背景執行
docker-compose up -d
```

### 5.3 開發除錯

```bash
# 直接透過 Maven 啟動 (含熱重載)
mvn spring-boot:run

# 執行測試
mvn test
```

## 六、匯入檔規則

| 規則 | 說明 |
|------|------|
| 檔名格式 | 前兩碼為數字 + 底線 + 報表名，如 `01_報表.xlsx` |
| 副檔名 | 必須為 `.xlsx` |
| 工作表 | 僅讀取第一個工作表 (Index 0) |
| 內容 | 包含樣板共用欄位 + 該公司獨有資料 |

## 八、檢核機制

| 檢核項目 | 觸發條件 | 行為 |
|---------|---------|------|
| 目錄不存在 | 指定年/月/報表路徑無效 | 跳過該任務，紀錄警告 |
| 樣板不存在 | templet/ 中無 .xlsx | 跳過該任務，紀錄警告 |
| 檔案數量不符 | 實際數 ≠ expected-file-count | 中止該報表，紀錄錯誤 |
| 檔名格式不符 | 前兩碼非數字 | 中止該報表，紀錄錯誤 |
| 座標重複寫入 | 兩份匯入檔寫入同一 cell | 中止該報表，刪除不完整輸出，紀錄錯誤 |

> 各報表為獨立事件，一份報表失敗不影響其他報表處理。

## 九、常見問題

**Q: 找不到樣板檔？**  
確認 `templet/` 目錄中有 `.xlsx` 檔案，且檔名不以 `~` 開頭 (Excel 暫存檔)。

**Q: 執行後沒有產出？**  
檢查 `config/application.yml` 的 `process-year` / `process-month` 設定是否正確，以及 `import/` 目錄結構是否符合規範。

**Q: 報告顯示重複座標？**  
表示兩份以上匯入檔在同一儲存格位置填入了資料。需排查來源檔案，確認資料歸屬。
